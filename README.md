# ChatGPT 对话保存助手（无附件版）

一个用于 **自动保存 ChatGPT 对话** 的油猴脚本，支持导出为 **HTML / Markdown / PDF / JSON**，并支持将之前导出的 **JSON 上下文重新导入到输入框**，方便继续历史对话。

---

## 功能特性

- 自动检测对话更新并自动保存
- 支持导出格式：
  - HTML
  - Markdown
  - PDF
  - JSON（上下文）
- 支持两种保存方式：
  - 浏览器默认下载目录
  - 指定本地文件夹（推荐）
- 支持恢复已授权的保存文件夹
- 支持将导出的 JSON 上下文重新导入 ChatGPT 输入框
- 支持手动一键导出
- 支持多工作区 / 多对话分类保存
- 适配：
  - `https://chat.openai.com/*`
  - `https://chatgpt.com/*`
  - `https://*.openai.com/*`
  - `https://*.chatgpt.com/*`

---

## 效果说明

安装后，页面右下角会出现一个悬浮按钮 **“存”**。

点击后会打开控制面板，你可以：

- 选择保存文件夹
- 切换回浏览器下载模式
- 勾选需要导出的格式
- 手动导出当前对话
- 导入之前保存的 JSON 上下文

当你使用“文件夹模式”时，脚本会在检测到新回复完成后自动保存，无需重复手动操作。

## 安装前准备

你需要先安装一个用户脚本管理器（油猴扩展），推荐使用：

- **Tampermonkey**

---

## 第一步：安装油猴（Tampermonkey）

### Chrome / Edge 安装方法

1. 打开浏览器扩展商店
2. 搜索 **Tampermonkey**
3. 找到官方扩展并点击安装
4. 安装完成后，浏览器右上角会出现 Tampermonkey 图标

常见入口：

- Chrome 网上应用店
- Edge 加载项商店

### Firefox 安装方法

1. 打开 Firefox 插件商店
2. 搜索 **Tampermonkey**
3. 安装扩展
4. 安装后确认插件已启用

---

## 第二步：安装本脚本

你可以使用以下任意一种方式安装。

### 方式一：直接从 GitHub 安装（推荐）

1. 打开本仓库中的脚本文件：

   `ChatGPT 对话保存助手（无附件版）.user.js`

2. 点击 **Raw**
3. 浏览器会自动唤起 Tampermonkey 安装页面
4. 点击 **安装**

### 方式二：手动新建脚本并粘贴

1. 点击浏览器中的 Tampermonkey 图标
2. 选择 **创建新脚本**
3. 删除默认内容
4. 将本项目里的完整脚本内容粘贴进去
5. 按 `Ctrl + S` 保存

---

## 第三步：打开 ChatGPT

安装成功后，打开以下任一站点：

- `https://chatgpt.com/`
- `https://chat.openai.com/`

页面加载完成后，右下角会出现悬浮按钮：

- **存**

如果出现提示：

- `ChatGPT 对话保存助手已加载`

说明脚本已成功运行。

## 快速开始

### 1. 打开控制面板

点击右下角悬浮按钮 **“存”**。

### 2. 选择保存方式

你有两种模式可选：

#### 模式 A：文件夹模式（推荐）

点击 **“选择文件夹”**，选择一个本地文件夹用于保存对话。

优点：

- 支持自动保存
- 文件按工作区 / 对话名称自动分类
- 不会每次都弹浏览器下载框
- 体验最好

> 建议专门新建一个文件夹，例如：
>
> `ChatGPT-Exports`

#### 模式 B：浏览器下载模式

如果不选择文件夹，脚本会使用浏览器默认下载目录。

特点：

- 适合临时导出
- 手动导出时会下载文件到浏览器默认下载位置
- **自动保存不会触发**，避免频繁弹出下载窗口

## 导出格式说明

你可以在面板中勾选需要导出的格式。

### HTML

适合：

- 保留接近网页阅读体验
- 本地归档
- 浏览器中直接打开查看

### Markdown

适合：

- 笔记整理
- Obsidian / Typora / VS Code 编辑
- 二次编辑与发布

### PDF

适合：

- 打印
- 存档
- 发给他人查看

### JSON Context

适合：

- 保存纯文本上下文
- 后续重新导入到 ChatGPT 输入框
- 延续历史对话内容

---

## 自动保存流程

当满足以下条件时，脚本会自动保存：

- 已启用自动保存（默认开启）
- 当前为 **文件夹模式**
- 已授权保存文件夹
- 当前对话有新增消息
- ChatGPT 已完成生成，页面内容稳定

脚本会自动判断当前对话是否需要更新，避免重复覆盖无变化内容。

在“文件夹模式”下，脚本会自动按以下结构保存：

```text
你选择的根目录/
└── 工作区名称/
    └── 对话标题/
        ├── _meta.json
        ├── html/
        │   └── 对话标题.html
        ├── md/
        │   └── 对话标题.md
        ├── pdf/
        │   └── 对话标题.pdf
        └── context/
            └── 对话标题.json
